# Drop-Point Method

The steps Claude follows. Same shape I use to read any source: load the real thing, load one example of good, compare, report the gap. No code required.

## Step 1: Build the timeline
Segment the transcript into moments. For each, capture the line or timestamp, who spoke, and a one-phrase summary. These coordinates get cited in every later step.

## Step 2: Trace the objection trail
List every hesitation, concern, or question the buyer raised. Mark each:
- **ANSWERED AND REOPENED:** handled, then confirmed later it landed.
- **ANSWERED ONCE:** handled in the moment, never returned to.
- **UNANSWERED:** raised and dropped.
Cite the line where it was raised, and where it was handled if it was.

## Step 3: Find the drop point
Locate the single line where buyer momentum fell and did not recover. Signals:
- A buyer question answered too fast, then the topic changed.
- Price stated without tying it to the buyer's own cost of staying stuck.
- Buyer talk share drops sharply after a specific line.
- A concrete next step that got asked for, then softened to "I'll follow up".
Name ONE drop point. Give the exact line or timestamp and quote the line.

## Step 4: Score the five reads
Apply `system/five-reads-rubric.md`. Score each read 0 to 5 against `context/best-closes.md`. For each read, name the one line the owner did better, so the gap is concrete.

## Step 5: Write the one follow-up line
One message the owner can send today. Rules:
- Open by naming the exact moment that dropped, in plain words.
- Reopen that one thread. Don't re-pitch the whole call.
- Tie back to the buyer's own stated cost or goal from earlier in the call.
- End with one low-friction question and a specific day offer.
- Match the owner's voice from `context/offer-and-voice.md`. Keep it under 60 words.
- No pressure language. One thread, one question.
